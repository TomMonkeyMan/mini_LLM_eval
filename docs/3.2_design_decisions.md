## P0-1: Provider 接口设计 ✅

### 决策总览

| 问题 | 决策 | 理由 |
|------|------|------|
| 返回结构 | 按生产标准设计，含 token_usage/cost/request_id | 用 Qwen 开发测试，没有的字段返回 None |
| 重试策略 | 指数退避 [1s, 2s, 4s] | 简单且业界惯例 |
| 可重试错误 | timeout/5xx/429/连接失败 可重试；4xx 不重试 | 区分临时故障和请求错误 |
| Mock 定位 | 无 API key 也能开发 + 单元测试可选 | 保证项目可独立运行 |
| Mock 模式 | 映射表为主，支持 fallback | 简单够用 |

### Provider 返回结构

```python
from dataclasses import dataclass
from typing import Optional

@dataclass
class TokenUsage:
    prompt_tokens: int
    completion_tokens: int
    total_tokens: int

@dataclass
class ProviderResponse:
    # 必填字段
    output: str
    latency_ms: float
    status: str                          # success | error | timeout

    # 可选字段（没有就返回 None）
    error: Optional[str] = None
    token_usage: Optional[TokenUsage] = None
    cost: Optional[float] = None
    model_name: Optional[str] = None
    request_id: Optional[str] = None
```

### Provider 基类

```python
from abc import ABC, abstractmethod

class BaseProvider(ABC):
    @abstractmethod
    async def generate(self, query: str, **kwargs) -> ProviderResponse:
        """生成响应"""
        pass

    @property
    @abstractmethod
    def name(self) -> str:
        """Provider 唯一标识"""
        pass

    async def health_check(self) -> bool:
        """健康检查（可选实现）"""
        return True

    async def close(self) -> None:
        """清理资源（可选实现）"""
        pass
```

### 重试策略

```python
RETRY_CONFIG = {
    "max_retries": 3,
    "retry_delays": [1, 2, 4],  # 指数退避，单位：秒
    "retryable_errors": [
        "timeout",
        "connection_error",
        "rate_limit",       # 429
        "server_error",     # 5xx
    ],
    "non_retryable_errors": [
        "bad_request",      # 400
        "unauthorized",     # 401
        "forbidden",        # 403
        "not_found",        # 404
        "invalid_response", # 响应解析失败
    ]
}
```

## P0-2: Evaluator 接口设计 ✅

### 决策总览

| 问题 | 决策 | 理由 |
|------|------|------|
| 接口参数 | 分开传 case_metadata 和 config | 职责清晰 |
| 同步/异步 | 统一同步 | MVP 都是规则类，LLM Judge 后期加 |
| 插件注册 | 待讨论 | - |
| 多 Evaluator | 支持，可配置策略 all/any | 灵活性 |

### Evaluator 返回结构

```python
from dataclasses import dataclass
from typing import Optional, Dict, List

@dataclass
class EvalResult:
    passed: bool
    reason: str                   # 人类可读的解释
    evaluator_type: str
    details: Optional[dict] = None
    error: Optional[str] = None   # evaluator 执行出错时记录
```

> **注**：MVP 不需要 score，只需 passed/failed。后续 LLM Judge 再加 score。

### Evaluator 基类

```python
from abc import ABC, abstractmethod

class BaseEvaluator(ABC):
    @abstractmethod
    def evaluate(
        self,
        output: str,
        expected: str,
        case_metadata: dict = None,    # case 级别信息
        config: dict = None            # evaluator 配置
    ) -> EvalResult:
        """执行评估"""
        pass

    @property
    @abstractmethod
    def name(self) -> str:
        """Evaluator 名称"""
        pass
```

### Case + Evaluator 配置层级

```
Case 级别定义 (eval_types)
    ↓ 如果没指定
全局默认配置 (config.yaml)
    ↓ 如果没配置
系统默认 (["contains"])
```

> **注**：不支持 CLI 运行时覆盖，所有配置落在数据集中，保证可追溯。

### 多 Evaluator 支持

**执行方式**：每个 evaluator 独立执行，结果独立存储，**不做合并**。

聚合统计是报表层职责，用数据库查询实现。

### EvalCase 数据结构

```python
from dataclasses import dataclass, field
from typing import List

@dataclass
class EvalCase:
    case_id: str
    query: str
    expected_answer: str
    tags: List[str]
    difficulty: str                # easy | medium | hard

    # Evaluator 配置：支持多个，每个独立执行
    eval_types: List[str] = field(default_factory=lambda: ["contains"])

    metadata: dict = field(default_factory=dict)
```

### CaseResult 数据结构

```python
from dataclasses import dataclass
from typing import Dict, Optional
from datetime import datetime

@dataclass
class CaseResult:
    case_id: str
    query: str
    expected: str
    actual_output: str

    # 每个 evaluator 的结果（独立存储，不合并）
    eval_results: Dict[str, EvalResult]  # {"contains": EvalResult, "regex": EvalResult}

    # Provider 相关
    latency_ms: float
    provider_status: str           # success | error | timeout
    error_message: Optional[str] = None
    retries: int = 0

    created_at: datetime = None
```

> **注**：不再有聚合的 passed/score，每个 evaluator 结果独立，聚合是报表层职责。

### CLI 使用示例

```bash
# evaluator 由 case 的 eval_types 决定，不支持运行时覆盖
python run_eval.py --dataset data/cases.jsonl --provider qwen
```

> 如需使用不同 evaluator，修改数据集文件或生成新数据集。
