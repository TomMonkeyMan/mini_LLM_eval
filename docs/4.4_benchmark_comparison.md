## 架构设计对比

### 1. 数据模型

```
┌─────────────────────────────────────────────────────────────────────┐
│                          数据模型对比                                │
├─────────────────────────────────────────────────────────────────────┤
│ Promptfoo:  TestCase { vars, assert[], options }                    │
│ DeepEval:   LLMTestCase { input, actual_output, retrieval_context } │
│ RAGAS:      SingleTurnSample { user_input, response, contexts }     │
│ Evals:      TaskState { task_description, messages, current_state } │
│ AgentProbe: AgentRecording { steps[], input, output }               │
└─────────────────────────────────────────────────────────────────────┘
```

### 2. Provider 抽象

| 框架 | 抽象方式 | 特点 |
|------|---------|------|
| **Promptfoo** | Interface + Registry | 前缀路由 `openai:chat:gpt-4o` |
| **DeepEval** | ABC + Factory | 懒加载、结构化输出 |
| **RAGAS** | ABC + Wrapper | LangChain 集成 |
| **Evals** | Protocol | 松耦合、可组合 |
| **AgentProbe** | Adapter 模式 | Monkey-patch 拦截 |

### 3. 指标实现

| 框架 | 实现模式 | 特点 |
|------|---------|------|
| **Promptfoo** | 函数注册表 | 59 个独立 handler |
| **DeepEval** | Schema+Template+Impl | 三文件分离 |
| **RAGAS** | Mixin 组合 | MetricWithLLM + MetricWithEmbeddings |
| **Evals** | Eval 类继承 | eval_sample() 模板方法 |
| **AgentProbe** | 断言链 | Fluent API |

### 4. 执行引擎

```
┌───────────────────────────────────────────────────────────────┐
│                     执行引擎对比                               │
├───────────────────────────────────────────────────────────────┤
│ Promptfoo:  并发 + 自适应速率限制 + Slot 队列                  │
│ DeepEval:   AsyncConfig + Semaphore + 超时控制                │
│ RAGAS:      Executor + 批量处理 + 进度条                       │
│ Evals:      串行为主，ThreadPoolExecutor 可选                  │
│ AgentProbe: 单次执行 + 录制回放                                │
└───────────────────────────────────────────────────────────────┘
```

## 选型建议

### 按场景选择

| 场景 | 推荐框架 | 理由 |
|------|---------|------|
| **RAG 系统评测** | RAGAS | 专用指标、异步优先 |
| **通用 LLM 评测** | DeepEval | Python 原生、48+ 指标 |
| **安全/Red Team** | Promptfoo | 69+ 插件、完善策略 |
| **大规模基准测试** | Evals | 465 评测、Registry 系统 |
| **Agent 行为测试** | AgentProbe | 执行追踪、断言框架 |
| **快速原型验证** | Promptfoo | YAML 配置、即开即用 |

### 按团队选择

| 团队特点 | 推荐框架 |
|---------|---------|
| **Python 为主** | DeepEval > RAGAS > Evals |
| **需要 Web UI** | Promptfoo > DeepEval |
| **CI/CD 集成** | DeepEval (pytest) > Promptfoo |
| **自定义指标多** | DeepEval > RAGAS |
| **Provider 多样** | Promptfoo (94) > DeepEval (18) |

## Mini LLM Eval 设计建议

基于以上分析，建议 Mini LLM Eval 采用以下设计：

### 核心架构

```python
mini_llm_eval/
├── config/
│   ├── schema.py        # Pydantic 配置模型 (参考 Promptfoo YAML)
│   └── loader.py        # YAML + Jinja2 加载
├── providers/
│   ├── base.py          # Protocol 定义 (参考 Evals)
│   ├── registry.py      # 前缀注册表 (参考 Promptfoo)
│   └── openai.py        # 实现示例
├── evaluators/
│   ├── base.py          # 装饰器注册 (参考 AgentProbe)
│   ├── exact_match.py
│   ├── contains.py
│   ├── similarity.py
│   └── llm_judge.py     # LLM-as-Judge (参考 DeepEval)
├── metrics/
│   ├── base.py          # Mixin 模式 (参考 RAGAS)
│   ├── faithfulness/    # 三文件模式 (参考 DeepEval)
│   │   ├── schema.py
│   │   ├── template.py
│   │   └── metric.py
│   └── context_precision.py
├── executor/
│   ├── runner.py        # 异步执行 (参考 RAGAS Executor)
│   ├── cache.py         # 缓存策略 (参考 Promptfoo/DeepEval)
│   └── rate_limiter.py  # 速率限制 (参考 Promptfoo)
├── output/
│   ├── models.py        # 结果模型
│   └── exporters.py     # JSON/CSV/Markdown
├── cli/
│   └── main.py          # Typer + Rich (参考 AgentProbe)
└── api/
    └── server.py        # FastAPI (可选)
```

### 优先级排序

#### P0 - 必须实现
1. Pydantic 配置模型 + YAML 加载
2. Provider Protocol + 注册表
3. 装饰器注册 Evaluator
4. 异步执行器 + Semaphore 限流
5. JSONL 事件记录
6. 缓存 (hash key 排除敏感信息)

#### P1 - 推荐实现
1. LLM-as-Judge (G-Eval 模式)
2. 三文件指标模式 (Schema+Template+Impl)
3. 成本追踪
4. Rich CLI 输出
5. 多格式导出 (JSON/CSV/Markdown)

#### P2 - 可选实现
1. Red Team 插件
2. Web UI
3. 多轮对话支持
4. HuggingFace 数据集集成
5. OpenTelemetry 追踪
