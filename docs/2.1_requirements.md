# Mini LLM Eval & Experiment Runner

一个最小可用的大模型评测运行系统，用于 LLM 微调、RAG、Agent 和工具调用系统的回归测试。

## 功能特性

- **评测任务管理**: 提交、执行、查询评测任务
- **多 Provider 支持**: Mock、OpenAI、Anthropic 等，可扩展
- **插件式 Evaluator**: exact_match、contains、regex、json_field、llm_judge
- **指标聚合**: 通过率、延迟分布、按 tag 聚合
- **实验对比**: 对比两次评测，识别回归和改进
- **状态机**: 任务状态流转与日志追踪
- **并发执行**: asyncio + Semaphore 限流

## 开放方向

- **A. 任务状态机**: PENDING → RUNNING → SUCCEEDED/FAILED/CANCELLED
- **B. 插件式 Evaluator**: 装饰器注册，自动发现
- **E. 并发与限流**: asyncio + Semaphore

## 技术栈

- **API**: FastAPI
- **数据库**: SQLite + SQLAlchemy (async)
- **并发**: asyncio + httpx
- **CLI**: typer + rich
- **日志**: structlog
- **测试**: pytest + pytest-asyncio

## API 接口

| Method | Endpoint | 描述 |
|--------|----------|------|
| POST | `/runs` | 创建评测任务 |
| GET | `/runs/{id}` | 查询任务状态 |
| GET | `/runs/{id}/cases` | 查询 case 结果 |
| POST | `/runs/{id}/cancel` | 取消任务 |
| POST | `/compare` | 对比两次评测 |
| GET | `/providers` | 列出 providers |
| GET | `/evaluators` | 列出 evaluators |

## AI 工具使用说明

- **使用的 AI 工具**: Claude Code
- **AI 辅助部分**: 架构设计讨论、代码框架生成、文档撰写
- **关键设计决定**:
  - 选择 FastAPI + SQLite 技术栈
  - 采用插件式注册机制
  - 使用 asyncio 处理并发
- **验证方式**: 代码审查、单元测试、集成测试
